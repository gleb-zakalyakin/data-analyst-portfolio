## Проекты

### 1. ETL‑пайплайн Яндекс Такси (Airflow + PySpark)

**Папка:** [`airflow-taxi-pipeline`](https://github.com/gleb-zakalyakin/data-analyst-portfolio/tree/main/airflow-taxi-pipeline)  

Учебный data engineering‑проект: ежедневный ETL‑пайплайн на Apache Airflow и PySpark агрегирует поездки Яндекс Такси по способам оплаты и формирует витрину `taxi_payment_summary` в ClickHouse.

**Что сделано:**
- чтение данных о поездках из Parquet (Object Storage / S3);
- расчёт количества поездок, среднего чека, средних чаевых и суммарной выручки по `payment_type`;
- DAG в Airflow: ожидание файла в S3 (`S3KeySensor`) и запуск PySpark‑задачи (`DataprocCreatePysparkJobOperator`).

### 2. A/B‑тест нового алгоритма рекомендаций (Python, pandas)

**Папка:** [`ab_testing_product`](https://github.com/gleb-zakalyakin/data-analyst-portfolio/tree/main/ab_testing_product)  

Анализ A/B‑теста в развлекательном приложении с бесконечной лентой контента. Цель — оценить влияние нового рекомендательного алгоритма на вовлечённость и конверсию сессий в «глубокий» просмотр.

**Что сделано:**
- проверка корректности сплита и очистка пересечений между группами;
- расчёт конверсии в успешные сессии и метрик активности;
- Z‑тест для долей и t‑тест Стьюдента для сравнения групп;
- показано, что доля «глубоких» сессий в тестовой группе статистически значимо выше (p‑value < 0.05).

### 3. Дашборд анализа заказов интернет-магазина (Apache Superset)

**Папка:** [`superset-orders-dashboard`](https://github.com/gleb-zakalyakin/data-analyst-portfolio/tree/main/superset-orders-dashboard)  

Аналитика интернет-магазина по данным о заказах за январь–май 2025 года. Дашборд построен для бизнес-пользователей для отслеживания выручки, динамики заказов и воронки статусов в интерактивном режиме.

**Что сделано:**
- очистка и предобработка данных на Python (обработка пропусков, аномалий, приведение типов) для 7 исходных таблиц;
- Feature Engineering (расчет категорий рейтинга);
- сборка дашборда в Apache Superset с KPI-плашками, динамикой выручки, распределением статусов;
- выявлен 2-кратный рост заказов к маю и определена зона риска бизнеса — 25% отмененных заказов.

### 4. Анализ паттернов поведения пользователей «Яндекс Книги» (ClickHouse)

**Папка:** [`clickhouse_yandex_books`](https://github.com/gleb-zakalyakin/data-analyst-portfolio/tree/main/clickhouse_yandex_books)  

Ad hoc анализ метрик потребления контента и паттернов использования мобильных приложений («Букмейт iOS/Android») с помощью аналитических функций ClickHouse. 

**Что сделано:**
- написание сложных SQL-запросов с использованием специфичных агрегаций ClickHouse (`sumIf`, `avgIf`, `uniqExact`, `argMax`), CTE и оконных функций;
- сегментация пользователей на «Слушателей» и «Читателей»;
- оценка популярности контента и контроль актуальных версий приложений;
- выявлено, что пользователи Android используют сервис более чем в 2 раза дольше пользователей iOS, но пользователи iOS практически не обновляют приложение (лишь 1.91% используют актуальную версию).

### 5. Анализ рынка недвижимости Санкт-Петербурга и Ленобласти (SQL)

**Папка:** [`SQL_ad_hoc_real_estate_market`](https://github.com/gleb-zakalyakin/data-analyst-portfolio/tree/main/SQL_ad_hoc_real_estate_market)  

SQL-проект по ad hoc-анализу архива объявлений Яндекс Недвижимости (2015–2018 гг.) для агентства, выходящего на новый рынок. Оценка ликвидности, сезонности и перспективных городов.

**Что сделано:**
- фильтрация аномалий и выбросов на уровне SQL (перцентили `PERCENTILE_DISC`);
- анализ сезонности публикации и снятия объявлений с помощью `EXTRACT`, `FILTER`, `CASE WHEN`;
- выявлены пиковые месяцы рыночной активности (февраль–апрель) и наиболее активные локации области (Мурино, Кудрово);
- составлена аналитическая записка с бизнес-рекомендациями по инвестициям в компактные объекты и выходу на рынок весной.

### 6. Многостраничный B2B‑дашборд продаж мед. пробирок (Yandex DataLens)

**Папка:** [`bi_datalens_b2b_sales`](https://github.com/gleb-zakalyakin/data-analyst-portfolio/tree/main/bi_datalens_b2b_sales)  

Многостраничный дашборд для отдела B2B‑продаж, который помогает отслеживать выручку, объём продаж и структуру ассортимента медицинских пробирок по регионам, категориям и продуктам.

**Особенности:**
- принцип **Overview → Detail**: вкладки «Обзор продаж», «Детализация» и «Аннотация»;
- матрица метрик: выручка, объём, средний чек, охват по регионам;
- глобальные фильтры: период, регион, тип клиента, канал продаж, поставщик, категория;
- аналитическая записка с выводами по драйверам роста (научные и клинические пробирки) и зонам риска (категория «Специализированные»).
