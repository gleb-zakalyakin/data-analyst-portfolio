## Проекты

### 1. ETL‑пайплайн Яндекс Такси (Airflow + PySpark)

**Папка:** [`airflow-taxi-pipeline`](https://github.com/gleb-zakalyakin/data-analyst-portfolio/tree/main/airflow-taxi-pipeline)  

Учебный data engineering‑проект: ежедневный ETL‑пайплайн на Apache Airflow и PySpark агрегирует поездки Яндекс Такси по способам оплаты и формирует витрину `taxi_payment_summary` в ClickHouse.[memory:95]

**Что сделано:**

- чтение данных о поездках из Parquet (Object Storage / S3);
- расчёт количества поездок, среднего чека, средних чаевых и суммарной выручки по `payment_type`;
- DAG в Airflow: ожидание файла в S3 (`S3KeySensor`) и запуск PySpark‑задачи (`DataprocCreatePysparkJobOperator`).[memory:95]

### 2. A/B‑тест нового алгоритма рекомендаций

**Папка:** [`ab_testing_product`](https://github.com/gleb-zakalyakin/data-analyst-portfolio/tree/main/ab_testing_product)  

Анализ A/B‑теста в развлекательном приложении с бесконечной лентой контента. Цель — оценить влияние нового рекомендательного алгоритма на вовлечённость и конверсию сессий в «глубокий» просмотр.[file:90][memory:96]

**Что сделано:**

- проверка корректности сплита и очистка пересечений между группами;
- расчёт конверсии в успешные сессии и метрик активности;
- Z‑тест для долей и t‑тест Стьюдента для сравнения групп;
- показано, что доля «глубоких» сессий в тестовой группе статистически значимо выше (p‑value \< 0.05).[file:90]

### 3. Многостраничный B2B‑дашборд продаж мед. пробирок (Yandex DataLens)

**Папка:** [`bi_datalens_b2b_sales`](https://github.com/gleb-zakalyakin/data-analyst-portfolio/tree/main/bi_datalens_b2b_sales)  

Многостраничный дашборд для отдела B2B‑продаж, который помогает отслеживать выручку, объём продаж и структуру ассортимента медицинских пробирок по регионам, категориям и продуктам.[memory:97]

**Особенности:**

- принцип **Overview → Detail**: вкладки «Обзор продаж», «Детализация» и «Аннотация»;
- матрица метрик: выручка, объём, средний чек, охват по регионам;
- глобальные фильтры: период, регион, тип клиента, канал продаж, поставщик, категория;
- аналитическая записка с выводами по драйверам роста (научные и клинические пробирки) и зонам риска (категория «Специализированные»).[memory:97]

### 4. Анализ монетизации и транзакций в игре (SQL, Gamedev)

**Папка:** [`sql_gamedev_monetization`](https://github.com/gleb-zakalyakin/data-analyst-portfolio/tree/main/sql_gamedev_monetization)  

Ad‑hoc анализ монетизации и транзакционной активности игроков по данным игры с премиальной внутриигровой валютой. Цель — понять, как игровые атрибуты персонажей связаны с конверсией в покупку и выручкой.[memory:92]

**Что сделано:**

- расчёт конверсии в платёж и когортный анализ по характеристикам персонажей;
- анализ транзакций (средний чек, медиана, разброс, распределение покупок);
- сравнение поведения платящих и неплатящих игроков;
- сводная витрина метрик (аудитория, конверсия, среднее число транзакций, AOV, ARPU);
- выявлен сегмент Northman с наибольшими значениями среднего чека и ARPU, сформированы продуктовые рекомендации.[memory:92]
