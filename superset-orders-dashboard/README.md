##  Описание проекта

Дашборд построен для анализа эффективности работы интернет-магазина: динамики заказов, выручки и конверсии по статусам. Инструмент позволяет бизнес-пользователям отслеживать ключевые показатели в интерактивном режиме.

**Период данных:** 01.01.2025 — 31.05.2025  
**Инструмент визуализации:** Apache Superset  
**Источник данных:** таблица заказов интернет-магазина

***

##  Структура данных

Файл `orders_filtered.csv` содержит **246 456 строк** и **5 колонок**:

| Поле | Тип | Описание |
|------|-----|----------|
| `order_id` | string (UUID) | Уникальный идентификатор заказа |
| `buyer_id` | string (UUID) | Идентификатор покупателя |
| `order_date` | datetime | Дата и время создания заказа |
| `status` | string | Статус заказа: `new`, `shipped`, `paid`, `canceled` |
| `total_amount` | float | Сумма заказа в рублях |

##  Подготовка данных

Перед построением дашборда данные прошли проверку и очистку.
Скрипт обработки доступен в файле [data_preprocessing.py](./data_preprocessing.py).

**Основные этапы:**
1. **Загрузка данных:** Импорт исходных таблиц из CSV-файлов (7 таблиц: `orders`, `order_items`, `products`, `users`, `categories`, `transactions`, `reviews`).
2. **Очистка (Data Cleaning):**
   - Проверка на дубликаты во всех таблицах — дубликаты не обнаружены.
   - Приведение дат к типу `datetime` (`order_date`, `review_date`, `transaction_date`, `registration_date`).
   - Исправление аномалий: обнулены отрицательные значения цены в таблице `products`.
   - Проверка пропущенных значений по всем таблицам.
3. **Feature Engineering:** Добавлена категория рейтинга (`High` / `Medium` / `Low`) в таблицу `reviews` на основе числового рейтинга.
4. **EDA:** Построены гистограммы и boxplot-диаграммы для числовых признаков, проанализирована корреляция между ценой и остатком товара.
5. **Фильтрация:** Для дашборда выгружен итоговый срез таблицы заказов за период **01.01.2025 — 01.06.2025** → `orders_filtered.csv` (246 456 строк).

##  Состав дашборда

Дашборд включает следующие визуализации:

- **KPI-плашки** — общее количество заказов, уникальные покупатели, суммарная выручка, средний чек
- **Линейный график** — динамика количества заказов и выручки по месяцам
- **Кольцевая диаграмма** — распределение заказов по статусам
- **Столбчатая диаграмма** — помесячное сравнение заказов по статусам
- **Таблица** — детализированный список заказов с фильтрацией

**Фильтры дашборда:**
- Период (диапазон дат)
- Статус заказа

##  Выводы по данным

- **Устойчивый рост:** количество заказов выросло более чем в **2 раза** с января по май — с 35 тыс. до 75 тыс. в месяц
- **Равномерное распределение статусов:** каждый статус занимает ~25%, что говорит о сбалансированной воронке
- **Высокий уровень отмен (25%)** — потенциальная точка роста для бизнеса
- **Взрывной рост в мае (+39%)** — возможно, связан с сезонностью или маркетинговыми активностями
