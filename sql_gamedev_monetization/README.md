# Анализ монетизации и транзакционной активности игроков (Gamedev)

##  Бизнес-задача
Оценить влияние пользовательских атрибутов (игровых характеристик персонажей) на конверсию в покупку премиальной внутриигровой валюты. Проанализировать транзакционную активность платящего сегмента аудитории и рассчитать ключевые продуктовые метрики монетизации (ARPU, ARPPU, средний чек, распределение транзакций).

##  Инструменты и стек
* **СУБД:** PostgreSQL
* **SQL:** 
  - Продвинутая агрегация с `FILTER (WHERE ...)`.
  - Математические и статистические функции: `PERCENTILE_DISC` (медиана), `STDDEV` (стандартное отклонение).
  - Подзапросы (Subqueries), `FULL JOIN` и `LEFT JOIN`.
  - Работа с типами данных и приведением типов (`NUMERIC`).

##  Краткое описание этапов работы
В рамках проекта реализованы SQL-запросы для вычисления следующих метрик:
1. **Общая конверсия:** расчет доли платящих игроков (Conversion Rate) по продукту в целом и когортный анализ в разрезе атрибутов персонажей.
2. **Анализ транзакций:** расчет базовых статистик по покупкам (количество, сумма, минимум, максимум, среднее, медиана).
3. **Сегментация пользователей:** сравнение поведенческой активности платящих (payer=1) и неплатящих (payer=0) пользователей, расчет среднего количества бесплатных транзакций.
4. **Анализ ассортимента:** определение долей самых востребованных игровых предметов во внутриигровом магазине.
5. **Сводная витрина метрик:** единый запрос (ad hoc), выводящий агрегированные данные: объем аудитории, конверсию в платеж, среднее количество транзакций на пользователя, средний чек (Average Order Value) и ARPU.

##  Главный вывод
«С помощью ad-hoc анализа базы данных и расчета перцентилей проведено исследование влияния игровых характеристик на монетизацию.

В результате выявлено, что общая конверсия в платеж распределена равномерно, однако сегмент Northman показал наивысшие значения среднего чека и ARPU среди всей аудитории при сохранении высокого уровня популярности (Volume).

Сформирована рекомендация: детально исследовать паттерны потребления этого сегмента с целью выявления драйверов высоких трат и последующего масштабирования успешных продуктовых механик на остальные сегменты пользователей»
